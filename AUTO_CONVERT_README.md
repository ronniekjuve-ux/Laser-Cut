# Автоконвертер DOC файлов через Word

## Проблема
LibreOffice (в Docker) не сохраняет кривые/дуги при конвертации DOC → HTML.
Microsoft Word конвертирует корректно.

## Решение
Скрипт `auto_convert.py` автоматически конвертирует DOC файлы через Word
при загрузке через веб-интерфейс.

## Запуск

### Вариант 1: Двойной клик
Запустите `start_converter.bat`

### Вариант 2: Из терминала
```bash
python auto_convert.py
```

## Как это работает

1. **Запустите** `auto_convert.py` (или `start_converter.bat`)
2. **Загрузите** DOC файл через веб-интерфейс (http://localhost:3000)
3. **Скрипт автоматически:**
   - Обнаружит новый файл в `data/uploads/`
   - Откроет его в Microsoft Word
   - Сохранит как HTML
   - Извлечет GIF изображения
   - Сохранит в `data/images/`
4. **Изображение появится** в интерфейсе

## Требования

- Microsoft Word (установлен)
- Python 3.8+
- pywin32: `pip install pywin32`
- watchdog: `pip install watchdog`

## Файлы

| Файл | Описание |
|------|----------|
| `auto_convert.py` | Основной скрипт автоконвертации |
| `start_converter.bat` | Бат-файл для запуска (Windows) |
| `convert.log` | Лог конвертации |

## Проверка

После загрузки DOC файла проверьте:
1. Появился ли GIF в папке `data/images/`
2. Отображается ли изображение в веб-интерфейсе
3. Проверьте лог: `convert.log`
