# Тестирование извлечения изображений из DOC

## Проблема
При парсинге DOC файлов раскладок некоторые линии пропадают на изображениях. Текущий метод конвертации DOC → HTML через LibreOffice на сервере (Docker/Linux) может некорректно отображать графические элементы.

## Решение
Созданы инструменты для локального тестирования на Windows машине, где установлен Microsoft Word.

## Варианты тестирования

### Вариант 1: Локальный скрипт (рекомендуется)

1. Установите pywin32 (если не установлен):
   ```bash
   pip install pywin32
   ```

2. Запустите скрипт:
   ```bash
   python test_extract_local.py samples/8мм001.Cnf.DOC
   ```

3. Результаты будут сохранены в папку `extracted_images/`

4. Проверьте изображения — есть ли все линии?

### Вариант 2: Детальный анализ

1. Запустите детальный скрипт:
   ```bash
   python test_extract_detailed.py samples/8мм001.Cnf.DOC
   ```

2. Скрипт сравнит три метода:
   - win32com → HTML
   - LibreOffice → HTML
   - win32com → PDF

3. Результаты сохранятся в `extracted_images_test/`

### Вариант 3: Через веб-интерфейс

1. Запустите фронтенд:
   ```bash
   cd frontend
   npm run dev
   ```

2. Откройте в браузере:
   ```
   http://localhost:3000/test-extract.html
   ```

3. Перетащите DOC файл в зону загрузки

4. Просмотрите результаты в браузере

## Файлы

| Файл | Описание |
|------|----------|
| `test_extract_local.py` | Простой скрипт для извлечения GIF |
| `test_extract_detailed.py` | Детальный анализ с сравнением методов |
| `frontend/public/test-extract.html` | Веб-интерфейс для тестирования |
| `app/api/v1/router_images.py` | Debug эндпоинты на сервере |

## Debug эндпоинты

### POST `/api/v1/images/debug/extract`
Загрузка DOC файла и извлечение изображений.

**Запрос:**
```bash
curl -X POST -F "file=@samples/8мм001.Cnf.DOC" http://localhost:8000/api/v1/images/debug/extract
```

**Ответ:**
```json
{
  "filename": "8мм001.Cnf.DOC",
  "html_size": 123456,
  "images_found": 3,
  "images": [
    {
      "name": "image1.gif",
      "size": 12345,
      "ext": ".gif"
    }
  ],
  "img_tags_in_html": 5,
  "img_tag_srcs": ["image1.gif", "image2.gif"],
  "html_preview": "<html>..."
}
```

### GET `/api/v1/images/debug/{filename}`
Извлечение изображений из уже загруженного файла.

**Пример:**
```bash
curl http://localhost:8000/api/v1/images/debug/8мм_layout_001_8мм001.Cnf.DOC
```

## Что проверять

1. **Количество изображений** — совпадает ли с ожидаемым?
2. **Целостность линий** — все ли линии отображаются на GIF?
3. **Качество** — нет ли размытия или артефактов?
4. **Сравнение методов** — какой метод (win32com vs LibreOffice) даёт лучший результат?

## Если линии пропадают

1. Проверьте PDF версию (метод 3 в детальном скрипте)
2. Сравните с оригинальным DOC файлом в Word
3. Попробуйте разные настройки экспорта
4. Проверьте версию LibreOffice (может требоваться обновление)

## Следующие шаги

После определения лучшего метода:
1. Обновить `unified_parser.py` для использования оптимального метода
2. Настроить параметры конвертации
3. Протестировать на разных типах раскладок
