# CLAUDE.md — Правила проекта Laser-Cut Backend

Эти правила ОБЯЗАТЕЛЬНЫ. Нарушение недопустимо без явного разрешения пользователя.

## Парсинг данных

- **Вес деталей — ТОЛЬКО из парсера DOC файла.** Никогда не пересчитывать по формуле `dx × dy × thickness × 7.85`. Парсер CypCut уже содержит правильный вес.
- **order_name не уникален.** В базе могут быть несколько заявок с одним order_name. Использовать `scalars().first()` вместо `scalar_one_or_none()` при поиске по order_name.

## Загрузка данных

- **Новая заявка = всегда новая запись.** При загрузке DOC файла создавать новую Application, не искать существующую по order_name. Мерж — отдельная операция.
- **DB varchar(50) enforced.** Всегда truncate перед вставкой.

## Имена и кодировка

- **normalize_name** должен обрабатывать Cyrillic↔Latin: `мм→mm`, `шт→sht`.

## Фронтенд

- **Axios ответы уже распарсены.** Никогда не `JSON.parse()` на `res.data`.
- **isRealMobile паттерн:** `isMobile && window.innerWidth <= 768`.

## Деплой

- **Два окружения:** GitHub + локальный Docker.
- **Quick deploy:** `docker cp <file> <container>:/app/<path>` + очистка `__pycache__`.

## Перед изменением кода

- **Всегда grep перед удалением переменной** — если переменная используется, удаление вызовет ReferenceError.

## SQLAlchemy async

- **Lazy loading в async = greenlet_spawn error.** Всегда использовать `selectinload`/`joinedload` при обращении к relationship в async контексте.
- **API limit:** Значение `le=200` в FastAPI Query блокирует запросы с `limit>200` (ошибка 422). Увеличивать при необходимости.

## Пагинация

- **Клиентская пагинация** для заказов: загружать все данные одним запросом, пагинация на фронте (15 на страницу). Страницы всегда полные (кроме последней).
- **Поиск** работает по всем данным до пагинации.
