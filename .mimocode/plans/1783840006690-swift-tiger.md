# План: Складской учёт материалов

## Краткое описание

Внедрение полноценного складского учёта металла с автоматическим списанием листов при резке заявок, журналом движения материалов и связью "заявка ↔ склад".

---

## 1. Модель данных

### 1.1 Обновить `WarehouseItem` (`app/db/models.py`)

Добавить поля:
- `sheet_w: Float` — ширина листа (мм)
- `sheet_h: Float` — высота листа (мм)
- `min_quantity: Integer, nullable` — минимальный остаток (для будущих оповещений)
- `last_deducted_at: DateTime(tz), nullable` — дата последнего списания

Убрать/оставить для совместимости:
- `size: String(50)` — оставить, но генерировать автоматически как `"{sheet_w}x{sheet_h}"`

### 1.2 Новая модель `WarehouseMovement` (`app/db/models.py`)

Журнал движения материалов:
- `id: Integer PK`
- `warehouse_item_id: FK -> warehouse_items`
- `application_id: FK -> applications, nullable`
- `quantity_change: Integer` — +/- сколько списано/приписано
- `movement_type: String(20)` — 'deduction', 'return', 'manual_adjustment', 'initial'
- `reason: Text, nullable` — причина
- `created_by: FK -> users, nullable`
- `created_at: DateTime(tz), server_default=now()`

### 1.3 Обновить `Application` (`app/db/models.py`)

Добавить поля:
- `warehouse_item_id: FK -> warehouse_items, nullable` — привязка к позиции на складе
- `sheets_used: Integer, nullable` — сколько листов списано
- `warehouse_deducted: Boolean, default=False` — было ли списание

## 2. API Endpoints

### 2.1 Обновить `router_warehouse.py`

| Метод | Путь | Описание |
|-------|------|----------|
| PATCH | `/api/v1/warehouse/{item_id}` | Обновить позицию (metal, grade, sheet_w, sheet_h, sheet_count, owner, note) |
| POST | `/api/v1/warehouse/{item_id}/deduct` | Списание: body `{quantity, application_id?}` |
| POST | `/api/v1/warehouse/{item_id}/return` | Возврат: body `{quantity, application_id?, reason}` |
| GET | `/api/v1/warehouse/{item_id}/movements` | История движения по позиции |
| GET | `/api/v1/warehouse/balance` | Текущие остатки с фильтрами (metal, grade, sheet_w, sheet_h) |

### 2.2 Обновить `router_applications.py`

| Метод | Путь | Описание |
|-------|------|----------|
| PATCH | `/api/v1/applications/{app_id}/warehouse` | Привязать заявку к складу: body `{warehouse_item_id, sheets_used}` |
| POST | `/api/v1/applications/{app_id}/deduct` | Списание листов при резке (auto + manual) |
| POST | `/api/v1/applications/{app_id}/cancel-deduct` | Отмена списания |
| GET | `/api/v1/applications/{app_id}/warehouse` | Инфо о привязке к складу |

## 3. Бизнес-логика

### 3.1 Автосписание при резке

Когда заявка получает статус `cut` (резка завершена):
1. Проверить `warehouse_item_id` и `sheets_used` на заявке
2. Если привязка есть — автоматически списать `sheets_used` листов
3. Создать запись `WarehouseMovement` с типом `deduction`
4. Уменьшить `sheet_count` на `warehouse_items`

### 3.2 Отмена списания

При отмене статуса `cut` (или возврате в `partially_cut`/`in_progress`):
1. Вернуть `sheets_used` листов на склад
2. Создать `WarehouseMovement` с типом `return`
3. Увеличить `sheet_count`

### 3.3 Ручное списание

Через PATCH `/api/v1/warehouse/{item_id}/deduct`:
1. Проверить наличие на складе
2. Списать указанное количество
3. Создать `WarehouseMovement`
