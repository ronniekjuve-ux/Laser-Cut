# План: Улучшение графика смен операторов

## Контекст

Текущий график смен работает на чистом фронтенде (hardcoded SHIFT_CYCLE + SHIFT_OFFSET). Требуется:
1. Сдвинуть OFFSET на 1 день (сегодня: Андрей + Денис → завтра: Андрей + Дима)
2. Ручные корректировки смен (отпуска, подмены) с сохранением долгосрочного чередования
3. Фильтр по оператору (чекбокс → подсветка + статистика за месяц)
4. Расчёт часов (12ч/день): отработано, предстоит, итого за месяц

## Дизайн-решения

- SHIFT_CYCLE и getShiftInfo выносятся в общий модуль `frontend/src/utils/shifts.js`
- Сдвиг OFFSET: today(20.06.2026) = "Andrey + Denis" (index 1) → `SHIFT_OFFSET = 3`
- Ручные корректировки хранятся в `localStorage` как Map `{ "YYYY-MM-DD": { st1: "Name", st2: "Name", night: "Name" | null } }`
- При наличии override для даты — используется он, иначе — расчётный из цикла
- Фильтр оператора: state `selectedOperator` → на календаре дни без него полупрозрачные, в сайдбаре — статистика за месяц
- Часы: день = 12ч, ночь = 12ч (20:00-08:00)

## Фаза 2: Ручные корректировки смен

### 2.1 Хранилище override-ов

**Ключ localStorage**: `shift_overrides`

**Формат**:
```json
{
  "2026-07-04": { "st1": "Dima", "st2": "Yura", "night": null },
  "2026-07-15": { "st1": "Andrey", "st2": "Andrey", "night": "Vova" }
}
```

### 2.2 Модификация `getShiftForDate(date, overrides)`

```js
export function getShiftForDate(date, overrides = {}) {
  const key = dateToKey(date); // "YYYY-MM-DD"
  if (overrides[key]) {
    return {
      pair: [overrides[key].st1, overrides[key].st2],
      isVovaOn: overrides[key].night === 'Vova',
      isOverride: true
    };
  }
  // fallback на цикл
  return { ...getShiftInfo(date), isOverride: false };
}
```

### 2.3 UI: Кнопка "Изменить смену" в сайдбаре

При клике на дату в сайдбаре — кнопка "✏️ Изменить смену". Открывает inline-форму с тремя dropdown:
- Станок 1 (день): [Yura, Denis, Andrey, Dima, ---нет---]
- Станок 2 (день): [Yura, Denis, Andrey, Dima, ---нет---]
- Ночь: [Vova, ---нет---]

Кнопки "Сохранить" и "Сбросить к расписанию".

## Фаза 3: Фильтр по оператору

### 3.1 Чекбоксы операторов

Над календарём — ряд чекбоксов: `[ ] Yura [ ] Denis [ ] Andrey [ ] Dima [ ] Vova`

При выборе оператора:
- На календаре: дни, где оператор НЕ работает — `opacity: 0.3`
- В сайдбаре: вместо "Детали дня" → "Статистика: [Имя оператора]"
- Кнопка "Сбросить фильтр"

### 3.2 Статистика оператора за месяц (сайдбар)

```
Andrey — Июнь 2026

Дней на станке: 14
Предстоит: 8 дней
Итого за месяц: 22 дня / 264 часа

Станок 1: 7 дней (84ч)
Станок 2: 7 дней (84ч)
```

### 3.3 Подсчёт часов

- 1 рабочий день = 12 часов (08:00-20:00)
- Ночная смена = 12 часов (20:00-08:00)
- `отработано` = дни от начала месяца до сегодня, где оператор в смене
- `предстоит` = дни от завтра до конца месяца, где оператор в смене
- `итого` = отработано + предстоит
