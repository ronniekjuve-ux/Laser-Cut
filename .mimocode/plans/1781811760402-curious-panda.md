# План: Перестройка логики Заявки → Заказы

## Цель
- **Вкладка Заявки** = предварительный просчёт (загрузка, просмотр данных, калькулятор стоимости, одобрение/отклонение)
- **Вкладка Заказы** = производственные заказы (текущий функционал Заявок: активные + выполненные)
- Одобрение заявки → автоматически появляется во вкладке Заказы
- Калькулятор стоимости — UI без сохранения (цены вводятся вручную)

## Архитектура
- Статус определяет вкладку: Заявки = `pending/rejected`, Заказы = `approved/in_progress/partially_cut/cut`
- Одна модель `Application` — нет миграции, `status` это `String(20)` в БД
- `router_orders.py` (отдельная модель Order) не трогаем — оставляем как есть

## Верификация

1. `docker-compose up --build` — все контейнеры стартуют
2. POST /auth/login → JWT
3. Upload .doc → заявка появляется во вкладке Заявки (status=pending)
4. Открыть заявку → виден калькулятор с данными резки
5. Ввести цены → автоматический пересчёт
6. Нажать «Утвердить» → заявка исчезает из Заявок, появляется в Заказах
7. Нажать «Отклонить» → заявка остаётся в Заявках сrejected
8. В Заказах: смена статуса (in_progress → partially_cut → cut)
9. Уведомления при одобрении/отклонении
10. Excel экспорт в обеих вкладках
11. Поиск и фильтры работают в обеих вкладках
