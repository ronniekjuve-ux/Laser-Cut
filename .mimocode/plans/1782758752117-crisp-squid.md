# План: Исправление деплоя Coolify — frontend build fails

## Проблема

Coolify не может собрать frontend. Ошибка:
```
exit code: 127 — npm not found
```

## Настройки Coolify

- Build Pack: Docker Compose
- Docker Compose Location: `/docker-compose.yml`

## Анализ

Coolify копирует код в `/artifacts/` и собирает оттуда. Ошибка `exit code: 127` для `npm run build` означает что npm не найден в контейнере.

Наш `docker-compose.yml` указывает `context: ./frontend`, но Coolify может:
1. Игнорировать context и собирать из корня
2. Использовать корневой Dockerfile (backend) вместо frontend

## Решение

### Добавить explicit dockerfile в build section

В `docker-compose.yml` для frontend добавить явный путь к Dockerfile:

```yaml
frontend:
  build:
    context: ./frontend
    dockerfile: Dockerfile
```

Это поможет Coolify найти правильный Dockerfile.

### Проверить .dockerignore в frontend/

Убедиться что `frontend/.dockerignore` не исключает нужные файлы.

## Файлы для изменения

- `docker-compose.yml` — добавить `dockerfile: Dockerfile` в build frontend

## Верификация

1. Запушить изменения
2. В Coolify нажать "Reload Compose File" → "Redeploy"
3. Проверить логи — npm должен быть найден
