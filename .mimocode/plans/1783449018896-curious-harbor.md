# Исправление веса деталей: всегда брать вес из DOC файла парсера

## Проблема
Вес детали показывается как 1264.961 кг вместо реальных 406.19 кг. Причина: код игнорирует вес из парсера и всегда пересчитывает по формуле `dx × dy × thickness × 7.85 / 1000000` (вес bounding box прямоугольника).

## Решение
Убрать ВСЕ формулы пересчёта веса. Всегда использовать `part_data.weight` из парсера. Если веса нет в файле — оставить `None`.

## Что исправлять — `app/api/v1/router_applications.py`

### 1. Строка 274-277 (загрузка раскладки)
```python
# Было:
part_weight = None
if thickness > 0 and part_data.dx > 0 and part_data.dy > 0:
    part_weight = round(part_data.dx * part_data.dy * thickness * 7.85 / 1000000, 4)

# Стало:
part_weight = part_data.weight
```

### 2. Строки 305-313 (пересчёт после обновления толщины) — УДАЛИТЬ
Блок целиком перезаписывает вес из парсера формулой. Удалить, вес уже установлен выше.

### 3. Строка 1037-1039 (мерж раскладок)
```python
# Было:
part_weight = None
if thickness > 0 and part_data.dx > 0 and part_data.dy > 0:
    part_weight = round(part_data.dx * part_data.dy * thickness * 7.85 / 1000000, 4)

# Стало:
part_weight = part_data.weight
```

## Верификация
1. Загрузить DOC файл заказа (нерж 50мм) заново
2. Проверить: вес детали = 406.19 кг (а не 1264 кг)
3. Проверить: вес листа = 1554.634 кг (не изменился)
4. Проверить: мерж раскладок сохраняет правильные веса
