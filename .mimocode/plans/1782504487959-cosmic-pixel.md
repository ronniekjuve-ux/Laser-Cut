# План: Исправление docker-compose для деплоя через Coolify

## Сравнение текущего docker-compose с предложенным вариантом

Предложенный вариант на рабочем столе решает 4 из 5 проблем, но оставляет несколько замечаний.

### Что предложенный вариант делает ПРАВИЛЬНО:
1. Убирает `ports` у frontend, db, redis — Coolify сам настраивает проксирование
2. Добавляет Coolify labels для frontend
3. Заменяет локальный volume `./data/images` на именованный `images_data`
4. Убирает экспозицию портов БД и Redis наружу

### Что НУЖНО ещё исправить:

#### 1. `--reload` в production (docker-compose.yml:16)
Убрать `--reload` из команды backend — это для dev, в production замедляет работу.

#### 2. Дублирование секретов (docker-compose.yml:24-25)
`DATABASE_URL` и `SECRET_KEY` дублируются: и в `.env`, и в `environment`. Нужно оставить ТОЛЬКО `.env`, убрать `environment` секцию.

#### 3. CORS захардкожен (app/main.py:26)
`allow_origins=["http://localhost:3000"]` — не будет работать в продакшене. Нужно читать из `FRONTEND_URL` env var.

## Изменения

### Файл 1: `docker-compose.yml`
- Убрать `--reload` из команды backend
- Убрать строки `DATABASE_URL=...` и `SECRET_KEY=...` из `environment`
- Оставить только `FRONTEND_URL` в `environment` (его нет в .env)
- Итого `environment` секция: только `FRONTEND_URL=https://laser-cut.pro`

### Файл 2: `app/core/config.py`
- Добавить `FRONTEND_URL: str = "http://localhost:3000"` в Settings

### Файл 3: `app/main.py`
- Заменить хардкод `allow_origins=["http://localhost:3000"]` на `settings.FRONTEND_URL`

### Файл 4: `.env`
- Добавить `FRONTEND_URL=http://localhost:3000` (значение по умолчанию для dev)
- **Примечание**: На сервере в Coolify нужно будет задать `FRONTEND_URL=https://laser-cut.pro`

### Файл 5: `entrypoint.sh`
- Убрать `--reload` (уже не нужен, т.к. убираем из команды тоже)

## Верификация
- `docker compose config` — проверить валидность compose-файла
- `docker compose build` — собрать образы
- `docker compose up -d` — запустить и проверить что все сервисы стартуют
