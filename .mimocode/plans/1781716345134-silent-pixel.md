## Ключевые недостатки LaserCut Core

### Критические (влияют на конкуренцию)

1. **Нет интеграции со станками**
   - Конкуренты: Bysoft, Tpm, Lantek — отправляют программы реза напрямую на станок
   - LaserCut: только парсинг файлов, ручная загрузка на станок
   - **Решение:** Интеграция через DNC-сервер или FTP/SSH на станок

2. **Нет оптимизации раскроя (nesting)**
   - Конкуренты: автоматически оптимизируют расположение деталей на листе
   - LaserCut: ручная загрузка раскладок
   - **Решение:** Интеграция с nesting-алгоритмом (DeepNest, NestFab)

3. **Нет CRM-воронки**
   - Конкуренты: Битрикс, AmoCRM — воронка от заявки до оплаты
   - LaserCut: только учёт заявок, нет продаж
   - **Решение:** Добавить воронку: Заявка → Расчёт → Согласование → Изготовление → Оплата → Доставка

4. **Нет мобильного приложения**
   - Конкуренты: все крупные.players имеют мобильные приложения
   - LaserCut: только веб
   - **Решение:** PWA (Progressive Web App) — быстрое решение

### Важные (ухудшают UX)

5. **Нет тёмной темы**
   - Современные UI требуют тёмную тему
   - **Решение:** CSS-переменные для тем

6. **Нет drag-and-drop загрузки файлов**
   - Современный UX требует drag-and-drop
   - **Решение:** React DnD или native HTML5 drag-and-drop

7. **Нет графиков/диаграмм**
   - Нет визуализации: загрузка станков, выработка операторов
   - **Решение:** Recharts или Chart.js

8. **Нет кастомных полей**
   - Заказчики могут хотеть свои поля (номер ТТН, дата поставки)
   - **Решение:** JSON-поле для кастомных данных

### Мелкие (качество кода)

9. **Нет типизации API-ответов**
   - Frontend не знает структуру ответов
   - **Решение:** TypeScript или JSDoc

10. **Нет unit-тестов парсеров**
    - Парсеры — критичный код, без тестов risky
    - **Решение:** pytest для парсеров

11. **Нет CI/CD**
    - Нет автоматических проверок при коммите
    - **Решение:** GitHub Actions

## Родовые преимущества LaserCut Core

1. **Уникальный парсинг** — ни один конкурент не парсит .DOC заявки автоматически
2. **Скорость внедрения** — 1 день vs месяцы для 1С/SAP
3. **Стоимость** — бесплатно vs десятки тысяч долларов
4. **Простота** — минимальная кривая обучения
5. **Специализация** — заточен под лазерную резку, а не общего назначения
6. **Современный стек** — FastAPI + React + Docker, легко поддерживать

## Итог

**LaserCut Core** — это **нишевый product** для малых предприятий лазерной резки. Его главное преимущество — парсинг .DOC файлов и скорость внедрения. Главный недостаток — нет CRM-воронки и интеграции со станками.

**Рекомендация:** Сфокусироваться на **Фазе 1** (CRM-функции) и **Фазе 2** (nesting, PWA). Это даст максимальный ROI за минимальное время. MES-функции (Фаза 3) — только если будет спрос от крупных клиентов.
