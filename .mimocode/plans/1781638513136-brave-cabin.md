# План: Вкладка "Аудит" — производственная аналитика для бухгалтера

## Цель
Страница "Аудит" — инструмент для бухгалтера: просмотр заявок по датам/заказчикам, экспорт в Excel, табель работы операторов.

## Структура страницы
Две вкладки внутри "Аудит":
1. **Заявки** — таблица заявок с агрегированными данными по раскладкам
2. **Операторы** — табель смен/часов за месяц

---

## Вкладка 1: Заявки

### Фильтры (toolbar)
- Период: "от" / "до" (date inputs)
- Заказчик (select, загружается из API)
- Сброс фильтров

### Группировка
- По умолчанию: группировка по дате поступления (месяц)
- Select: "По дате" / "По заказчику" / "Без группировки"
- Строка-заголовок группы: название + кол-во заявок + агрегаты (общая масса, общая длина реза)

### Столбцы основной таблицы
| Заказчик | Заявка | Дата | Материал | Толщ. | Дав.мат | Станок | Листов | Длина реза, м | Проколы | Масса дет., кг |

Все数值ные столбцы — агрегаты по всем раскладкам заявки (sum cut_length, sum pierces, sum weight).

### Данные
- Backend эндпоинт `GET /api/v1/applications/audit` — возвращает заявки с вложенными раскладками
- Frontend агрегирует: cut_length, pierces, weight — суммы по раскладкам

---

## Вкладка 2: Операторы

### Табель работы
- Таблица: Оператор | Дата смены | Смена (день/ночь) | Часы | Станок
- Фильтр по месяцу (select: Январь–Декабрь + год)
- Inline-редактирование: клик на ячейку "Часы" → input
- Кнопка "Сохранить" для изменённых строк
- Кнопка "Добавить смену" — новая строка

### Данные
- Новая модель `OperatorShift` в БД (или use existing `UserActivity`?)
- Backend CRUD: GET /api/v1/audit/operators, POST, PUT
- Привязка к существующим пользователям (User)

## Файлы для изменения

### Бэкенд

#### 1. `app/api/v1/router_applications.py` — эндпоинт `/audit`
```python
@router.get("/audit")
async def audit_applications(
    date_from: date | None = None,
    date_to: date | None = None,
    customer_id: int | None = None,
    db, user
):
```
- JOIN Application → Customer → ApplicationLayout → ApplicationLayoutPart
- Агрегация: sum(cut_length), sum(pierces), sum(weight) per application
- Ответ: list of enriched application dicts with layouts summary

#### 2. `app/api/v1/router_audit.py` — новый роутер для операторов
- `GET /api/v1/audit/operators?month=2026-06` — табель за месяц
- `POST /api/v1/audit/operators` — добавить смену
- `PUT /api/v1/audit/operators/{id}` — обновить часы
- `DELETE /api/v1/audit/operators/{id}` — удалить смену

#### 3. `app/db/models.py` — модель `OperatorShift`
```python
class OperatorShift(Base):
    __tablename__ = "operator_shifts"
    id, user_id (FK→users), date, shift_type ("day"/"night"),
    hours (float), machine_type (str), created_at
```

#### 4. `app/main.py` — подключить новый роутер
```python
from app.api.v1.router_audit import router as audit_router
app.include_router(audit_router, prefix="/api/v1")
```
