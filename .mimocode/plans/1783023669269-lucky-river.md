# План: Мобильные карточки для Заказов и Выполненных

## Проблема
Операторы работают исключительно через телефон. Текущая таблица заказов неудобна — столбцы обрезаются, приходится горизонтально скроллить, тяжело быстро ориентироваться.

## Решение
На мобильных экранах (`<=768px`) заменять таблицу на **вертикальный список карточек**. Каждая карточка — визуальная раскладка + основная информация. Тап по карточке открывает упрощённую модалку (без времени резки/проколов/перемещения).

## Что нужно изменить

### 1. Бэкенд: добавить данные первой раскладки в list API
**Файл:** `app/api/v1/router_applications.py` (строка ~463, enriched.append)

В ответ list API добавить поля первой активной раскладки:
```python
"layout_image": first_layout.layout_image if first_layout else None,
"sheet_size": f"{first_layout.sheet_w}x{first_layout.sheet_h}" if first_layout else None,
"sheet_count": sum(l.sheet_count or 1 for l in active_layouts),
```

Это нужно, чтобы карточки могли показывать изображение раскладки без отдельного запроса на каждую заявку.

### 2. Фронтенд: компонент MobileOrderCard
**Новый файл:** `frontend/src/components/MobileOrderCard.jsx`

Карточка для мобильного отображения:
- **Верхняя часть:** изображение раскладки (`layout_image`) на всю ширину карточки, с aspect-ratio 16:10
- **Нижняя часть:** 
  - Заказчик (жирный)
  - Строка: материал + толщина
  - Строка: размер листа + кол-во листов
  - Бейдж статуса (В очереди / В резке / Частично вырезано / Вырезано)
  - Бейдж группы (если есть)
  - Индикатор давальческого материала

### 3. Фронтенд: модалка детали для оператора (MobileOrderDetail)
**Новый файл:** `frontend/src/components/MobileOrderDetail.jsx`

Упрощённая версия ApplicationDetail для мобильного:
- Изображение раскладки сверху
- Информация: заказчик, материал, толщина, размер листа, вес
- Группа (если есть)
- Заметки (с возможностью редактирования)
- Давальческий материал (бейдж)
- **Статус** — выпадающий список для admin/operator (в очереди / в резке / частично вырезано / вырезано)
- Прогресс по листам (квадратики, как сейчас)
- **Список деталей** — название, размеры (DX x DY), кол-во, изображение (при тапе)
- **НЕ включать:** время резки, время перемещения, время проколов, длину резки, перемещение в мм, кол-во проколов

## Данные для карточки

| Поле | Источник | Пример |
|------|----------|--------|
| Изображение | `layout_image` (из list API) | `/api/v1/images/layouts/...` |
| Заказчик | `customer` | "РВС-2000" |
| Материал | `steel_grade \|\| material` | "ст3" |
| Толщина | `thickness` | "8 мм" |
| Размер листа | `sheet_size` (новое) | "1500x3000" |
| Кол-во листов | `sheet_count` (новое) | 3 |
| Статус | `status` | "В резке" |
| Группа | `group_name` | "окантовка" |
| Дав. мат | `supply_material` | Да/Нет |

## Данные для модалки (дополнительно к карточке)

|------|----------|------------|

## Верификация

1. `docker compose up --build -d frontend backend`
2. Открыть на телефоне (или DevTools с emulation 375px)
3. **Заказы:** проверить что карточки рендерятся, изображения видны, статусы правильные
4. **Тап по карточке:** модалка открывается с деталями, детали с изображениями, статус меняется
5. **Выполненные:** аналогично, проверить кнопку "Вернуть в заказы"
6. **Desktop:** убедиться что таблица как раньше, без изменений
7. Пагинация работает на обеих вкладках
